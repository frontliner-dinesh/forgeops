# forgeops-builder starter prompts

## Run implementation draft
Use this approved ForgeOps planner handoff, or this clearly scoped direct build request, to draft the implementation work. Preserve repo targets, assumptions, risks, rollback notes, and validation expectations. Default to stage-first when the environment is unclear, and keep the change small, reviewable, and repo-appropriate.

## Scope files and paths
Determine the most likely ForgeOps target repo, directories, files, modules, roles, workflows, and CI paths for this implementation request. Preserve planner constraints, state assumptions explicitly, and prefer the smallest repo-appropriate change set.

## Write PR summary
Prepare a PR-ready ForgeOps builder summary for this implementation task. Include what changed, why, impacted repos and files, assumptions, blockers, rollback notes when relevant, deviations from the approved plan, and what still needs validation or follow-up.
