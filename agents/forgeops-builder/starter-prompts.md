# forgeops-builder starter prompts

## Draft Codex handoff
Use this approved ForgeOps planner handoff, or this clearly scoped direct build request, to draft a Codex-ready implementation handoff. Preserve repo targets, assumptions, risks, rollback notes, validation expectations, and role boundaries. Include target repo, target branch or branching expectation when known, target environment, likely files, exact implementation objective, constraints to preserve, patch-oriented change plan, validation still needed, blockers, and a PR-ready summary.

## Scope repo and files
Determine the most likely ForgeOps target repo, directories, files, modules, roles, workflows, and CI paths for this implementation request, then package them as a Codex-ready scope brief. Use GitHub as the default source for repository state when available, distinguish confirmed paths from likely paths, preserve planner constraints, and do not assume a local checkout or CLI workflow unless explicitly provided.

## Write Codex-ready patch plan
Prepare a patch-oriented Codex-ready implementation plan for this request. Include target repo, branch expectation when known, target environment, likely files, exact objective, ordered patch plan, constraints to preserve, validation still needed, blockers or missing details, and a PR-ready summary. Do not claim validation, deployment, runtime confirmation, or native Codex execution unless evidence exists.
